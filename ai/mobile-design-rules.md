# Mobile Demo Design Rules

本文件是开发 `/mobile-demo` 前的移动端设计护栏。移动端可以重新排版桌面模块，但不得新增业务数字、改写 mock 数据，所有数据点必须能追溯到现有桌面 mock 数据或 `ai/mobile-data-contract.json`。

## 1. 设计目标

移动端 demo 定位为企业内部风险管理对话式机器人，延续当前桌面版的产品气质：

- 浅米白页面背景
- 白色内容卡片
- 橙色作为唯一主强调色
- 浅灰边框
- 16-20px 圆角
- 轻阴影
- 稳重、清晰、温暖，不做深色科技大屏

移动端页面按 390px 宽度设计，后续实现时必须优先保证数据一致、信息层级清楚、无横向溢出。

## 2. 视觉规则

- 禁止深色科技大屏风格。
- 禁止渐变大背景。
- 禁止密集小字堆叠。
- 禁止把桌面表格直接缩小塞进手机。
- 页面横向宽度按 390px 设计。
- 页面左右 padding 固定为 16px。
- 正文字号不得小于 13px。
- 关键数字字号控制在 20-28px。
- 卡片圆角控制在 16-20px。
- 卡片间距控制在 12-16px。
- 主色只使用橙色，辅助风险色只用于风险状态。
- 单个页面不超过 4 种强调色：橙色、绿色、黄色、红色；灰色只作为中性色。
- 所有主要内容必须放在白色卡片中，边框使用浅灰。
- 不使用大面积装饰图、复杂背景纹理或消费级聊天 App 风格。

## 3. 信息密度规则

每个移动端模块必须包含 4 层，不得省略信息层级：

1. AI 摘要卡：一句话讲结论，只保留最关键判断。
2. 核心指标卡：最多展示 4 个关键数字，其他指标进入详情。
3. 主分析区：使用一个图表、横向条、信息流或折叠列表。
4. 明细区：使用 accordion 或 bottom sheet 展开，不直接铺满大表。

桌面端表格迁移到移动端时必须先判断字段重要性：

- 主字段进入卡片标题或主值。
- 次要字段进入辅助行。
- 明细字段进入 accordion 或 bottom sheet。
- 找不到数据时显示“暂无数据”，不得编造。

## 4. 交互规则

移动端只允许使用以下交互：

- 顶部返回
- segmented tabs
- accordion
- bottom sheet
- 横向指标卡
- 查看详情
- sticky bottom AI input

禁止新增以下内容：

- 复杂交互或多层业务流程
- 无意义动画
- 与桌面版无关的新页面
- 未在桌面版出现的数据入口
- 真实接口请求、登录鉴权、权限流程

## 5. 数据规则

- 所有金额、比例、评级、日期、名单数量、舆情数量必须来自现有 mock 数据或 `ai/mobile-data-contract.json`。
- 不允许新增金额、比例、评级、日期、名单数量、舆情数量。
- 不允许改写已有数字的精度、单位和正负方向。
- 移动端空间不够时可以隐藏字段，但不能改字段。
- 隐藏字段必须可通过详情、accordion 或 bottom sheet 承接。
- 找不到数据时统一显示“暂无数据”。
- 每个模块开发前必须先对照 `ai/mobile-redesign-inventory.json` 和 `ai/mobile-data-contract.json`。

## 6. 组件规则

移动端业务页面只能基于以下基础组件搭建：

- `MobileShell`
- `MobileHeader`
- `MobileSummaryCard`
- `MobileMetricCard`
- `MobileSegmentedTabs`
- `MobileDataCard`
- `MobileAccordion`
- `MobileBottomSheet`
- `MobileRiskBadge`
- `MobileStickyInput`

组件职责边界：

- 基础组件只负责结构、样式和轻量交互。
- 基础组件不得内置业务数字。
- 基础组件不得直接读取 mock 数据。
- 业务页面负责传入来自 desktop mock 或 data contract 的展示值。
- 新增业务组件前必须确认现有基础组件无法覆盖。

## 7. 模块拆页约束

每个桌面模块移动端拆页时，应先按以下顺序收敛信息：

1. 结论优先：先展示 AI 摘要卡。
2. 指标优先：最多 4 个关键指标。
3. 走势或分布优先：主分析区只放一个核心视图。
4. 明细后置：表格字段进入 accordion 或 bottom sheet。

桌面图表转换建议：

- 趋势柱状图：转为紧凑柱状图或横向条。
- 维度柱状图：转为横向条。
- 堆叠柱状图：转为分组卡片或可展开横向条。
- 大表格：转为折叠列表。
- 详情表：转为 bottom sheet。
- 多 tab 页面：使用 segmented tabs，不新增多级 tab。

## 8. 验收规则

每完成一个移动端模块，必须截图并完成自评：

- 是否存在横向溢出
- 是否与桌面 mock / data contract 数字一致
- 是否存在小于 13px 的正文
- 是否卡片过密
- 是否乱编金额、比例、评级、日期、数量
- 是否把桌面表格硬塞进手机
- 是否保留 AI 摘要、核心指标、主分析区、明细区四层结构

未通过自评的模块不得进入下一模块。
